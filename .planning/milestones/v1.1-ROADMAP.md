# Milestone v1.1: Polish & Feel

**Status:** ✅ SHIPPED 2026-04-13
**Phases:** 4-8
**Total Plans:** 5

## Overview

Elevated the v1.0 game with smooth animations, persistence, sound, and system dark mode — making it feel like a finished product. All six polish features were delivered: CSS-only theming via `prefers-color-scheme`, localStorage score persistence, pop-in piece animations, computer thinking delay, synthesized Web Audio sound effects with mute toggle, and an animated win line through all 8 orientations.

Notable pattern: three of five phases (4, 7, 8) were verification-only — Phase 3 had pre-implemented all requirements in commit `18a87a0`. Only phases 5 and 6 required code changes.

**Key decision:** Phase 4 is verification-only — Phase 3 pre-implemented THEM-01, THEM-02, PERS-01, PERS-02 (commit `18a87a0`)

### Phase 5: CSS Piece Animations

**Goal**: Piece placement feels responsive and tactile with pop-in animation
**Depends on**: Phase 4
**Requirements**: ANIM-01, ANIM-03
**Plans**: 1 plan

Plans:

- [x] 05-01: Patch renderBoard() for incremental updates + human browser verification

**Details:**

Root cause: `innerHTML=''` full-wipe on every `renderBoard()` call recreated all 9 cells, triggering the pop-in animation on existing pieces. Fix: incremental DOM update pattern — builds 9 cells once, then only adds `cell--x`/`cell--o` to cells whose state changed from empty. `resetGame()` explicitly clears `boardEl.innerHTML` before `renderBoard()` to trigger a full rebuild. ANIM-03 (`prefers-reduced-motion`) was already satisfied by construction — animation gated on `@media (prefers-reduced-motion: no-preference)`.

**Key decision:** Incremental DOM update via `children.length !== 9` guard; CSS animation was already correct — no `style.css` changes needed

### Phase 6: Thinking Delay

**Goal**: The computer feels deliberate — it pauses before responding, like it's thinking
**Depends on**: Phase 5
**Requirements**: FEEL-01, FEEL-02
**Plans**: 1 plan

Plans:

- [x] 06-01: Fix FEEL-02 timer cancellation + human verification of delay behaviour

**Details:**

Replaced non-cancelable `thinkDelay()` helper with a module-level `thinkingTimer` variable storing the `setTimeout` ID. Added `clearTimeout(thinkingTimer)` at the start of `resetGame()` — prevents ghost moves on a freshly reset board. Added `game.get_status() !== 'playing'` guard post-delay as a safety net. FEEL-01 (300–800ms randomized) was already implemented in Phase 3; only FEEL-02 (cancellation) required a code fix.

**Key decision:** `clearTimeout` pattern (not AbortController or game-ID counter) — simpler and sufficient per established pitfall documentation

### Phase 7: Sound Effects & Mute

**Goal**: The game has synthesized audio feedback for all game events, with a persistent mute option
**Depends on**: Phase 6
**Requirements**: AUDI-01, AUDI-02, AUDI-03, AUDI-04, AUDI-05, AUDI-06, AUDI-07
**Plans**: 1 plan

Plans:

- [x] 07-01: Automated spot-checks + manual browser verification of pre-implemented sound effects and mute toggle

**Details:**

Web Audio OscillatorNode synthesizer with 5 distinct sounds (440Hz sine for human move, 330Hz sine for computer, 3-note fanfare for win, sawtooth for loss, 360Hz for draw). Lazy `AudioContext` singleton created on first `playTone()` call (inside user gesture) satisfies Chrome/Safari autoplay policy. Gain envelope (`exponentialRampToValueAtTime`) prevents audio click/pop artifacts. `osc.stop(time + duration)` schedules GC cleanup to prevent long-session audio graph growth. Mute toggle with `localStorage` persistence. Phase was verification-only — all 7 AUDI requirements pre-implemented in Phase 3.

**Key decision:** OscillatorNode synthesizer over audio files — zero network requests, no asset loading, ~82 lines of JS generating all sounds

## Milestone Summary

**Key Decisions:**

- Verify-first pattern: three phases (4, 7, 8) were verification-only because Phase 3 pre-implemented their requirements — confirmed via static analysis + human browser approval
- CSS-only theming: `:root` dark defaults + `@media (prefers-color-scheme: light)` overrides — no JS, no FOUC, no user toggle needed
- Incremental DOM update pattern (Phase 5): build once, patch later — avoids CSS animation re-trigger on untouched DOM nodes
- Cancelable timer via `clearTimeout` (Phase 6): module-level `thinkingTimer = null`, assigned on setTimeout, cleared in resetGame()
- Web Audio synthesis over audio files (Phase 7): zero bytes shipped, no network, ~82 LOC for all game sounds
- Lazy AudioContext (Phase 7): context created only on first user gesture — satisfies browser autoplay policy universally

**Issues Resolved:**

- `diag-rl` win-line CSS positioning — fixed `left: 95% + rotate(135deg)` anchor (Phase 4)
- Phase 3 feature work uncommitted — committed in Phase 4 finalization as `feat(03-01)` commit `18a87a0`
- Piece re-animation on every renderBoard() call — fixed with incremental DOM update in Phase 5
- Ghost computer move after New Game click — fixed with `clearTimeout` in Phase 6 (commit `524c53e`)

**Issues Deferred:**

None — all v1.1 scope delivered.

**Technical Debt Incurred:**

None identified — clean implementation with no known shortcuts.
