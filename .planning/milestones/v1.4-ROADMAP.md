# Milestone v1.4: Difficulty Levels

**Status:** ✅ SHIPPED 2026-04-28
**Phases:** 13–14
**Total Plans:** 2

## Overview

Added configurable difficulty levels to the Rust AI and exposed them through a player-facing dropdown in the browser UI. The AI's mistake rate is now parameterized by level: Easy (65%), Medium (25%), Hard (8%), Unbeatable (0%). The selected difficulty persists across page refreshes via localStorage and resets the game board immediately on change.

## Milestone Summary

**Key Decisions:**

- `set_difficulty(level: u8)` WASM API — u8 eliminates silent NaN/Infinity coercion at the JS→Rust boundary; rate mapping stays in Rust (not a JS concern)
- `mistake_rate_for_level(u8) -> f64` match table — explicit arms for all 4 levels; `_ => 0.25` wildcard handles future out-of-range values without panic
- `reset()` does NOT touch `difficulty` — setting persists across game resets (player picks level once, not before every game)
- `difficultyEl.disabled` mirrors `isProcessing` at exactly 1 set-true and 4 set-false sites — covers all async exit paths
- Change handler order: `set_difficulty(level)` → `saveDifficulty(level)` → `resetGame()` — WASM updated before reset so new level applies from move one
- `ttt-difficulty` localStorage key (not `ttt-score`) — follows existing naming convention, avoids corrupting score data
