## Directory Purposes

**`src/` (primary source — all hand-written code lives here):**
- Purpose: Both Rust game engine and JavaScript/CSS frontend
- Notable: Rust and JS/CSS files coexist in the same flat directory — no subdirectory split
- Key files: `lib.rs` (Rust library root), `main.js` (JS entry), `style.css` (all styles)

**`pkg/` (generated — wasm-pack output):**
- Purpose: WASM binary + JS glue generated by `wasm-pack build --target web --release`
- Generated: Yes — regenerate with `wasm-pack build --target web --release` from project root
- Committed: No (in `.gitignore`)
- Consumed by: `src/main.js` via `import init, { WasmGame } from '../pkg/tic_tac_toe.js'`
- Must exist before running `npm run dev`

**`dist/` (generated — Vite production bundle):**
- Purpose: Final static files served in production (nginx in Docker)
- Generated: Yes — by `npm run build`
- Committed: No (in `.gitignore`)
- Contains: `index.html`, hashed `assets/*.js`, `assets/*.css`, `assets/*.wasm`

**`target/` (generated — Rust build artifacts):**
- Generated: Yes — by `cargo build` / `wasm-pack build`
- Committed: No (in `.gitignore`)

**`.planning/` (planning artifacts):**
- Purpose: GSD workflow documents, phase plans, research
- Committed: Yes
- Key: `.planning/codebase/` — codebase analysis documents loaded by GSD commands

## Key File Locations

**Entry Points:**
- `index.html`: Browser HTML shell — loaded first, references `src/main.js` as ES module
- `src/main.js`: JS entry point — `main()` async function initializes WASM and wires all events
- `src/lib.rs`: Rust library crate root — declares `pub mod board`, `pub mod ai`, `pub mod wasm_api`
- `src/main.rs`: CLI binary entry — `fn main()` for terminal gameplay (separate from browser)

**Game Logic:**
- `src/board.rs`: `Game` struct, `Player` enum, `GameStatus` enum, move validation, win detection
- `src/ai.rs`: `get_computer_move()` — minimax algorithm with 25% random mistake rate
- `src/wasm_api.rs`: `WasmGame` struct — the sole WASM-exported type; translates Rust → JS primitives

**Frontend:**
- `src/main.js`: Event handlers, DOM rendering, score management, async computer-turn flow
- `src/audio.js`: Sound effects (`sounds.move()`, `sounds.win()`, etc.) + mute toggle
- `src/style.css`: All visual styles, CSS custom properties, animations, win-line positioning
- `index.html`: Static DOM skeleton (board, scoreboard, status, restart button)

**Configuration:**
- `Cargo.toml`: Rust deps (`wasm-bindgen`, `rand`, `getrandom`, `console_error_panic_hook`); `crate-type = ["cdylib", "rlib"]`
- `vite.config.js`: Vite WASM plugin, `build.target = 'esnext'`, `server.fs.allow = ['.']` for `pkg/` access
- `package.json`: `"type": "module"`, dev/build/preview scripts, Vite deps

**Build & Deploy:**
- `Dockerfile`: Multi-stage: `rust:slim` → wasm-pack → Vite build → `nginx:alpine` serve
- `nginx.conf`: Production nginx config for the static site
- `.github/workflows/docker.yml`: CI pipeline — Docker build and push

**Testing:**
- `test.html`: Browser WASM bridge smoke test — open at `/test.html` during `npm run dev`
- `src/board.rs` (lines 155–381): 11 `#[cfg(test)]` Rust unit tests via `cargo test`
- `src/ai.rs` (lines 153–339): 4 Rust unit tests including 100-game simulation

## Naming Conventions

**Rust files:**
- `snake_case.rs` for all modules: `board.rs`, `ai.rs`, `wasm_api.rs`, `lib.rs`, `main.rs`
- Module names match file names exactly (e.g., `pub mod wasm_api` in `lib.rs` → `src/wasm_api.rs`)

**Rust types:**
- Structs/enums: `PascalCase` — `WasmGame`, `Game`, `Player`, `GameStatus`
- Functions/methods: `snake_case` — `get_computer_move`, `make_move`, `update_status_for`
- Constants: `SCREAMING_SNAKE_CASE` — `WIN_LINES`, `MISTAKE_RATE`

**JavaScript:**
- Files: `camelCase.js` — `main.js`, `audio.js`
- Functions: `camelCase` — `renderBoard`, `handleCellClick`, `handleGameOver`, `resetGame`
- Constants: `SCREAMING_SNAKE_CASE` for fixed values — `PLAYER_X`, `PLAYER_O`, `NO_MOVE`, `SCORE_KEY`, `THINK_MIN`
- DOM variables: descriptive `camelCase` with `El` suffix for element refs — `boardEl`, `statusEl`, `restartBtn`, `winLineEl`

**CSS classes:**
- BEM-style: `block`, `block__element`, `block--modifier`
- Block examples: `.board`, `.cell`, `.status-message`, `.scoreboard`, `.win-line`
- Modifiers: `.cell--taken`, `.cell--x`, `.cell--o`, `.cell--winning`, `.board--disabled`
- State modifiers on status: `.status--win`, `.status--loss`, `.status--draw`, `.status--computer-turn`
- Win-line positional variants: `.win-line--row0`, `.win-line--col1`, `.win-line--diag-lr`, etc.

**WASM glue (generated — `pkg/`):**
- File naming: `tic_tac_toe.js`, `tic_tac_toe_bg.wasm` (crate name with underscores replacing hyphens)

## Where to Add New Code

**New Rust game logic (rules, state):**
- Extend `src/board.rs` — add to `Game` impl or add new types in the same file
- Expose to JS: add method to `WasmGame` in `src/wasm_api.rs` with `#[wasm_bindgen]`
- Rebuild: `wasm-pack build --target web --release` to regenerate `pkg/`

**New AI behavior:**
- Modify `src/ai.rs` — `get_computer_move()` or `minimax()`
- No WASM API changes needed unless adding new exported methods

**New UI feature:**
- JS logic: add to `src/main.js` or create a new `src/feature.js` module and import it
- Styles: add to `src/style.css` following BEM class naming
- HTML structure: add to `index.html`
- Audio: add new sound to the `sounds` object in `src/audio.js`

**New WASM export:**
- Add `pub fn` to `WasmGame` impl in `src/wasm_api.rs` with `pub` visibility and `#[wasm_bindgen]` on the impl block
- Run `wasm-pack build --target web --release`
- The new method appears automatically in `pkg/tic_tac_toe.js` and `pkg/tic_tac_toe.d.ts`

**New Rust unit tests:**
- Add `#[cfg(test)] mod tests { ... }` block to the relevant `src/*.rs` file
- Run: `cargo test`

**New planning documents:**
- Codebase analysis: `.planning/codebase/`
- Phase plans: `.planning/phases/`
- Research: `.planning/research/`

## Build Workflow (Dev)

```bash
# 1. Build Rust → WASM (must run first, or after any src/*.rs change)
wasm-pack build --target web --release
# Output: pkg/ directory

# 2. Start Vite dev server (hot-reload for JS/CSS; pkg/ served as-is)
npm run dev
# Vite serves index.html, src/main.js, src/style.css, and pkg/*.js/*.wasm

# 3. Manual WASM bridge tests (optional)
# Open http://localhost:5173/test.html in browser

# 4. Run Rust unit tests (no WASM needed)
cargo test
```
