## Color

| Role | Dark value | Light value | Usage |
|------|-----------|------------|-------|
| Dominant (60%) | `#1a1a2e` (`--bg`) | `#f0f2f7` (`--bg`) | Page background |
| Secondary (30%) | `#16213e` (`--surface`) | `#ffffff` (`--surface`) | Cell backgrounds, scoreboard fill, **`<select>` background** |
| Accent (10%) | `#e94560` (`--accent`) | `#e94560` (`--accent`) | See reserved list below |
| Text primary | `#e0e0e0` (`--text`) | `#1a1a2e` (`--text`) | Body text, **`<select>` foreground text** |
| Text dim | `#888` (`--text-dim`) | `#666` (`--text-dim`) | **`.difficulty-label`**, score labels, status dim text |

Accent (`--accent`) reserved for:

1. Game title text (`.title`)
2. Board gap lines (`.board` background color)
3. O-piece glyph color (`.cell--o`)
4. Winning cell highlight (`.cell--winning` background)
5. Win line overlay (`.win-line` background)
6. Restart button background (`.restart-btn`)
7. Focus ring on interactive elements (`.cell:focus-visible`, `.mute-btn:focus-visible`)

The `.difficulty-select` and `.difficulty-label` do NOT use `--accent`. Source: CONTEXT.md D-02, D-03.

Destructive: No destructive action exists in this phase (difficulty change resets unconditionally with no confirmation dialog). Source: CONTEXT.md D-05, D-06.

- `1px solid var(--text-dim)` — thin, theme-consistent border matching the `--text-dim` value in both dark and light modes.
- No `appearance: none` — browser-native dropdown arrow and rendering preserved. Source: CONTEXT.md D-03.

### Disabled State

- `.difficulty-select:disabled` — browser-native opacity (typically 0.5) + `cursor: not-allowed`.
- No custom class or override needed. Source: CONTEXT.md D-04.

### HTML Structure (exact)

```html
<div class="title-row">
  <h1 class="title">Tic-Tac-Toe</h1>
  <button class="mute-btn" id="mute-btn" aria-label="Toggle sound" aria-pressed="false">🔊</button>
  <span class="difficulty-label">Difficulty:</span>
  <select class="difficulty-select" id="difficulty-select" aria-label="Select difficulty">
    <option value="0">Easy</option>
    <option value="1" selected>Medium</option>
    <option value="2">Hard</option>
    <option value="3">Unbeatable</option>
  </select>
</div>
```

Notes:

- `<span>` and `<select>` are appended as the last two children of `.title-row`.
- No new wrapper element. The existing `display:flex; align-items:center; gap:12px` on `.title-row` handles alignment. Source: CONTEXT.md D-01.
- `value="1"` (Medium) is the default `selected` option. Source: REQUIREMENTS.md UI-03 + ROADMAP.md success criterion 3.
- `aria-label="Select difficulty"` on the `<select>` ensures the label is accessible even when the visible `<span>` is not programmatically associated. Implementer may alternatively use `<label for="...">` + remove the `<span>`, but the `<span>` + `aria-label` pattern is acceptable.

## Interaction Contract

### Difficulty Change Handler

Trigger: `change` event on `.difficulty-select`

Sequence (in order):

1. `parseInt(difficultyEl.value, 10)` → u8 integer 0–3
2. `game.set_difficulty(level)` — WASM call before reset
3. `saveDifficulty(level)` — persist to `localStorage['ttt-difficulty']`
4. `resetGame()` — unconditional board reset

Source: CONTEXT.md D-05, D-06, code_context "Established Patterns".

### Disabled State Synchronization

`difficultyEl.disabled` mirrors `isProcessing` at these exact points:

| Event | `isProcessing` | `difficultyEl.disabled` |
|-------|---------------|------------------------|
| `handleCellClick()` — player moves | `true` | `true` |
| Post-thinking-delay — computer finishes | `false` | `false` |
| `resetGame()` — board reset | `false` | `false` |
| Page load / `main()` startup | `false` | `false` |

Source: CONTEXT.md D-07.

### localStorage Persistence

| Key | Default | Type |
|-----|---------|------|
| `ttt-difficulty` | `1` (Medium) | String — parsed with `parseInt(value, 10)` |

Pattern: try/catch wrapping identical to existing `loadScore()` / `saveScore()` in `src/main.js`. Source: CONTEXT.md code_context.

### Startup Sequence

1. Query `#difficulty-select` → `difficultyEl`
2. `loadDifficulty()` → level (0–3, default 1 on first visit or parse error)
3. `game.set_difficulty(level)`
4. `difficultyEl.value = String(level)`

## States

| State | `.difficulty-select` appearance | Trigger |
|-------|--------------------------------|---------|
| Enabled, idle | Normal — `var(--surface)` bg, `var(--text)` text, `1px solid var(--text-dim)` border | Default |

No animation or transition on the disabled toggle — instantaneous state change via `difficultyEl.disabled` boolean. Source: CONTEXT.md D-04.

## Accessibility

- `.difficulty-label` is a visible `<span>`, not a `<label>` element. The `<select>` carries `aria-label="Select difficulty"` to cover screen readers.
- The `<select>` is a native form control — keyboard-navigable and screen-reader-accessible by default.
- Focus ring: not overridden on `<select>` — browser default focus indicator applies. No `:focus-visible` rule needed (native `<select>` has reliable OS-level focus styling).
- Disabled state announced by screen readers via the native `disabled` attribute (no additional aria needed).
