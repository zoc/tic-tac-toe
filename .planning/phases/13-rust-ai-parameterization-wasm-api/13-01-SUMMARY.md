# Phase 13 Plan 01: Rust AI Parameterization & WASM API Summary

**One-liner:** Added `mistake_rate_for_level(u8) -> f64` to ai.rs with 4-level match table, extended `get_computer_move` with `difficulty: u8` param, and exposed `set_difficulty(&mut self, level: u8)` through the WASM boundary with a persistent `difficulty: u8` field on `WasmGame`.

## What Was Built

Parameterized the Rust AI's mistake rate by difficulty level and exposed the setting via the WASM boundary so JavaScript can call `game.set_difficulty(level)` before `game.computer_move()`.

### Task 1: Parameterize src/ai.rs

- Removed `MISTAKE_RATE: f64 = 0.25` constant
- Added `mistake_rate_for_level(level: u8) -> f64` with match arms: 0→0.65, 1→0.25, 2→0.08, 3→0.0, `_→0.25` (wildcard for future-proof safety)
- Extended `get_computer_move(game: &Game, difficulty: u8) -> Option<usize>` signature
- Updated mistake injection: `rng.random_bool(mistake_rate_for_level(difficulty))`
- Updated all 7 test call sites to pass `difficulty=1`
- Added `test_mistake_rate_for_level` asserting all 4 level mappings
- Added `test_ai_unbeatable_never_loses` (50 games at `difficulty=3`, zero X wins)
- Commit: `d05a05d`

### Task 2: Extend src/wasm_api.rs and update src/main.rs

- Added `difficulty: u8` field to `WasmGame` struct (not exposed as getter)
- `WasmGame::new()` defaults `difficulty` to `1` (Medium)
- Added `set_difficulty(&mut self, level: u8)` — pure setter, no board reset
- `computer_move()` now passes `self.difficulty` to `get_computer_move`
- `reset()` left unchanged (does NOT touch `difficulty` — persists across games)
- `src/main.rs` call site updated to `get_computer_move(&game, 1)` (hardcoded Medium)
- Commit: `e3f37b5`

### Task 3: Verification

- `cargo test` — 22 tests pass (7 AI tests + 15 board tests), 0 failures
- `wasm-pack build --target web` — exits 0, pkg/ updated with new WASM binary and JS bindings
- `set_difficulty` confirmed exported: `wasm.wasmgame_set_difficulty(this.__wbg_ptr, level)` in tic_tac_toe.js
