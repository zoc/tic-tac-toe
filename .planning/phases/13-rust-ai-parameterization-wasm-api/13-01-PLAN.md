<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| JS → WASM (`set_difficulty`) | Untrusted JS sends a u8 level value into Rust game state |
| JS → WASM (`computer_move`) | Reads `self.difficulty` set by prior boundary crossing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Tampering | `WasmGame.difficulty` via `set_difficulty(level: u8)` | accept | u8 type guarantees value is 0–255; any out-of-range level (4–255) hits the `_ => 0.25` wildcard arm in `mistake_rate_for_level`, silently falling back to Medium. No panic, no invalid state. The valid range (0–3) is documented in the function's doc comment. |
| T-13-02 | Elevation of Privilege | AI difficulty state | accept | `difficulty` field is not exposed as a getter — JS can only write via `set_difficulty`, never read back. Consistent with the `inner: Game` field. No privilege escalation path exists in a single-player game. |
| T-13-03 | Information Disclosure | `difficulty` field visibility | accept | The `difficulty` field carries no sensitive data — it is a game preference, not a secret. No getter is exposed. Low-value target. |
| T-13-04 | Denial of Service | `random_bool(0.0)` for level=3 | accept | Verified in rand-0.10.1 source: `Bernoulli::new(0.0)` stores `p_int = 0`, always returns false. No panic, no infinite loop. minimax terminates because the board is finite (9 cells). |

**Residual risks:** None at high severity. All threats accepted or mitigated at the type level by Rust's u8 constraint and the wildcard match arm.
</threat_model>
