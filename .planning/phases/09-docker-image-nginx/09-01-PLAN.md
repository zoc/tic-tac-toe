**Change 1 — Replace curl|sh wasm-pack install with pinned cargo install (D-11, D-12):**

Replace this line in the build stage:
```dockerfile
RUN curl https://rustwasm.github.io/wasm-pack/installer/init.sh -sSf | sh
```

With:
```dockerfile
RUN cargo install wasm-pack@0.14.0 --locked
```

This eliminates the supply chain risk of fetching and executing an external script at build time.

**Change 2 — Replace apt Node.js with NodeSource Node 20 LTS (D-13, D-14):**

Replace the apt install block in the build stage:
```dockerfile
RUN apt-get update && apt-get install -y --no-install-recommends \
    curl \
    nodejs \
    npm \
    && rm -rf /var/lib/apt/lists/*
```

With:
```dockerfile
RUN apt-get update && apt-get install -y --no-install-recommends curl ca-certificates \
    && curl -fsSL https://deb.nodesource.com/setup_20.x | bash - \
    && apt-get install -y --no-install-recommends nodejs \
    && rm -rf /var/lib/apt/lists/*
```

This installs Node 20 LTS explicitly instead of whatever version `apt` provides (typically Node 18 on Debian bookworm).

**Change 3 — Add curl to serve stage and add HEALTHCHECK (D-01, D-02, D-03, D-04):**

In the serve stage, after `FROM nginx:alpine AS serve`, add these two lines before the COPY commands:
```dockerfile
# Install curl for HEALTHCHECK probe
RUN apk add --no-cache curl
```

And after `EXPOSE 80`, add the HEALTHCHECK instruction:
```dockerfile
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
    CMD curl -f http://localhost/healthz || exit 1
```

Final serve stage should look like:
```dockerfile
FROM nginx:alpine AS serve

# Install curl for HEALTHCHECK probe
RUN apk add --no-cache curl

# Replace the default nginx site config with our custom one

# Copy only the production build output
COPY --from=build /app/dist /usr/share/nginx/html

EXPOSE 80

HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
    CMD curl -f http://localhost/healthz || exit 1
```

  <action>
Make three targeted additions to nginx.conf. The existing structure has a `server { }` block with security headers, then `location /` and `location /assets/` and `location = /favicon.ico` blocks. Do not modify /assets/ block.

**Addition 1 — gzip block in the server block (D-05, D-06, D-07, D-08):**

Add this block inside the `server { }` block, after the security headers and before the first `location` block:

```nginx
    # Gzip compression for text assets — skip WASM (already compressed by wasm-opt)
    gzip on;
    gzip_vary on;
    gzip_min_length 1000;
    gzip_comp_level 6;
    gzip_types text/html text/css application/javascript text/javascript;
```

Note: `application/wasm` is intentionally omitted — WASM files are pre-compressed by wasm-opt during build; double-gzip wastes CPU for no benefit.

**Addition 2 — Cache-Control: no-cache for location / (D-09):**

In the existing `location / { }` block, add `add_header Cache-Control "no-cache";` so it reads:

```nginx
    location / {
        try_files $uri $uri/ /index.html;
        add_header Cache-Control "no-cache";
    }
```

**Addition 3 — /healthz endpoint (D-01, D-02):**

Add a new location block before `location = /favicon.ico`:

```nginx
    # Dedicated health check endpoint — lightweight, no body, no HTML
    location = /healthz {
        access_log off;
        return 200 "ok";
        add_header Content-Type text/plain;
    }
```

The `return 200 "ok"` directive serves a minimal text response. `access_log off` keeps health check noise out of container logs.
