<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → JS | Mute preference loaded from storage on page start — could be manually modified |
| User gesture → AudioContext | Audio only starts after user interaction (autoplay policy boundary) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | localStorage ttt-muted value | accept | Mute preference is cosmetic UI state — attacker gaining "sound on" or "sound off" has zero security impact |
| T-07-02 | Denial of Service | AudioContext node accumulation | mitigate | playTone() uses exponentialRamp decay + oscillator.stop(); nodes disconnect when finished. onended cleanup prevents long-session audio graph growth |
| T-07-03 | Elevation of Privilege | Web Audio API fingerprinting | accept | AudioContext can be used for audio fingerprinting by third parties — not applicable here (no third-party scripts; all code is first-party) |
</threat_model>
